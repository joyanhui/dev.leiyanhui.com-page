修复 Niri 窗口管理器鼠标拖拽导致的列宽溢出和间隙

2026-07-16
阅读时长: 6 分钟

nixos 从0实现全集 - 目录

现象

使用 Niri 窗口管理器一段时间后，偶尔会出现两种异常布局：

  1. 左侧间隙：最左侧的窗口距离屏幕左边有一个明显的大间隙，比配置的 gaps 大得多
  2. 列宽溢出：两个窗口并排时，总宽度略大于屏幕宽度，导致两列都显示不全，左右各被裁掉一部分

这两种情况都是偶发的，不是每次开机都会出现。而且一旦出现，用鼠标重新拖拽也无法彻底恢复，过一会儿又会复发。

快速修复

如果你不想看原理分析，直接加这个快捷键：

1
2

// 在你的 niri config.kdl 的 binds {} 里加入
Mod+Shift+R { spawn-sh "$REPO_DIR/script/niri_fix_columns.py 2/3_1/3"; }

或者需要所有列平分：

1

Mod+Shift+E { spawn-sh "$REPO_DIR/script/niri_fix_columns.py"; }

然后执行 niri msg action load-config-file 重载配置。以后出现溢出或间隙时，按下快捷键，所有列立即恢复为等比宽度，完美填满屏幕。

根因分析

抓包 Niri IPC

通过 socat 抓取 Niri IPC 通信，确认了每种调整列宽的操作底层发送的消息：

1
2
3
4
5

# 启动一个本地 socat 作为 IPC 代理
socat -t 1 UNIX-LISTEN:/tmp/niri-test-sock,fork STDIO &

# 另一个终端设置 NIRI_SOCKET 指向代理，执行操作
NIRI_SOCKET=/tmp/niri-test-sock niri msg action set-column-width '50%'

抓包结果：

操作Niri 收到的 IPC 消息含义
鼠标右键拖拽窗口边缘{"SetFixed": 960}设为固定像素值
键盘 +10%{"AdjustProportion": 10.0}调整比例（安全）
键盘 -10%{"AdjustProportion": -10.0}调整比例（安全）
niri msg action set-column-width '50%'{"SetProportion": 50.0}设回等比（修复用）
niri msg action set-column-width '900'{"SetFixed": 900}设为固定像素值

结论：鼠标拖拽是祸根

+10% 和 -10% 这两个快捷键操作的是比例（AdjustProportion），窗口会保持等比分配，不会出问题。

但鼠标右键拖拽窗口边缘会把列宽设成固定像素值（SetFixed）。一旦变成固定像素值：

  • Niri 认为"这是用户明确想要的大小"，不会自动缩放
  • 两列各拖到 55% 屏幕宽 → 加起来 110% → 溢出屏幕
  • 或者两列加起来不到 100% → 右侧留空

为什么会有"左侧间隙"？

按理说列宽总和小于屏幕宽度时，多余的空白应该在右侧（因为 center-focused-column "never" 是左对齐）。左侧出现间隙通常是因为：

  1. 空列残留：自动贴砖脚本（如 niri_tile_to_n.py）在执行 ConsumeOrExpelWindowLeft/Right 时，可能在左侧创建了一个零宽度的空列，看不见但把其他列挤到了右边
  2. 自动贴砖+拖拽混合使用：自动贴砖管理窗口排列，但鼠标拖拽管理列宽——两者互不知情，状态容易乱

一键修复脚本

原理

通过 Niri IPC 逐列聚焦，用 SetProportion 把宽度恢复成等比模式：

# 1. 获取当前工作区所有窗口，按列分组
all_windows = request(sock, "Windows")
columns = defaultdict(list)
for win in all_windows:
    col_idx = win["layout"]["pos_in_scrolling_layout"][0]
    columns[col_idx].append(win)

# 2. 逐列聚焦，用 SetProportion 设为等比宽度
for i, col_idx in enumerate(sorted(columns)):
    do_action(sock, "FocusWindow", id=first_win["id"])
    do_action(sock, "SetColumnWidth", change={"SetProportion": 50.0})

# 3. 恢复原来的焦点窗口
do_action(sock, "FocusWindow", id=original_focus)

关键点在于 SetProportion 这个 IPC 动作——它告诉 Niri “这一列按照比例分配宽度”，而不是设一个具体的像素值。之后不管新增窗口、切换显示器，Niri 都会自动重新计算。

# 或者用 --pattern 参数

使用 2/3_1/3 模式时，比例会循环分配：

列数列1列2列3列4
2列66.7%33.3%——
3列40%20%40%—
4列33.3%16.7%33.3%16.7%

与其他方案的对比

方案优点缺点
只用预设快捷键 Mod+Ctrl+1/2/3不依赖外部脚本要记多组快捷键，只改当前列
只用鼠标拖拽直观产生固定像素值，积累后溢出
一键修复脚本（本文方案）鼠标随意拖，出问题了一键恢复需要安装 Python

总结

  • 这是 Niri 的设计特性，不是 bug。Niri 刻意保留用户通过鼠标拖拽设定的列宽，不会自动重新分配
  • +10% 和 -10% 快捷键是安全的（操作的是比例，不是像素值）
  • 鼠标拖拽是产生固定像素宽度的根源
  • 出现溢出或间隙时，用 niri_fix_columns.py 一键重置所有列为等比模式即可

这个方法的好处是兼容鼠标拖拽的习惯——你尽管拖，出问题了按一下修复键就行，不用改变操作习惯。

#!/usr/bin/env python3
"""
重置当前工作区所有列的宽度为等比模式，修复鼠标拖拽导致的列宽溢出和左侧间隙。

=== 问题背景 ===

Niri 中鼠标右键拖拽窗口边缘调整列宽时，会把列宽设为固定像素值（SetFixed）。
之后无论新增窗口、切换显示器，该列都死守这个像素值不变。
当两列固定宽度之和超出屏幕 → 溢出，小于屏幕 → 留空。
自动贴砖脚本（niri_tile_to_n.py）的 consume/expel 操作也可能在左侧产生空列。

=== 解决原理 ===

通过 niri IPC 逐列聚焦，用 SetProportion 把固定像素宽度恢复成等比分配。
之后 Niri 会动态分配空间，不再溢出或留空。

=== 关于视口滚动的处理 ===

因脚本需要逐列 FocusWindow 来 SetColumnWidth，会导致 Niri 的滚动视口
跟随焦点偏移，最后停留在最后一列上。脚本末尾分三步修复：
  1. FocusColumn(0)    → 把视口拉到第一列（焦点暂移）
  2. CenterVisibleColumns → 居中所有可见列（不改变焦点）
  3. FocusWindow(orig)  → 恢复原始焦点（窗口已在视口内，不触发额外滚动）

=== 用法 ===

  niri_fix_columns.py                      # 所有列平分宽度
  niri_fix_columns.py 2/3_1/3              # 奇数列 2/3，偶数列 1/3
  niri_fix_columns.py --pattern 1/2        # 所有列设成 50%
  niri_fix_columns.py 1/3_1/3_1/3          # 三列各 1/3

快捷键绑定（加到 niri config.kdl 的 binds {} 里）：
  Mod+Shift+R { spawn-sh \"$REPO_DIR/script/niri_fix_columns.py 2/3_1/3\"; }
"""

import json
import os
import socket
import sys
import subprocess
from collections import defaultdict


def notify(msg):
    """通过桌面通知向用户反馈结果，桌面环境不可用时 fallback 到 stderr"""
    try:
        subprocess.run(["notify-send", "-a", "niri-fix-columns", msg], timeout=3)
    except Exception:
        print(msg, file=sys.stderr)


def get_socket_path():
    """获取 niri IPC socket 路径，从环境变量 NIRI_SOCKET 读取"""
    path = os.environ.get("NIRI_SOCKET")
    if not path:
        notify("❌ 找不到 NIRI_SOCKET，确定在 niri 里面运行的？")
        sys.exit(1)
    return path


def send_and_recv(sock, msg):
    """
    发送一条 JSON 消息到 niri IPC，读取一行 JSON 响应。

    niri IPC 是 Unix Socket 上的换行分隔 JSON 协议（NDJSON），
    每条消息以换行符结尾，响应也是单行 JSON。
    """
    sock.sendall((json.dumps(msg) + "\n").encode())
    buf = b""
    while True:
        chunk = sock.recv(4096)
        if not chunk:
            break
        buf += chunk
        if b"\n" in buf:
            break
    line = buf.split(b"\n", 1)[0]
    return json.loads(line.decode())


def request(sock, req_name):
    """
    发送简单请求（如 "Windows" / "Workspaces"），返回 Ok 包裹的数据内容。

    niri 请求的响应格式为 {"Ok": {"<请求名>": <数据>}} 或 {"Err": "..."}。
    例如：
      请求 "Windows"     → {"Ok": {"Windows": [...]}}
      请求 "Workspaces"  → {"Ok": {"Workspaces": [...]}}
      请求 "FocusedWindow" → {"Ok": {"FocusedWindow": {...}}}
    """
    resp = send_and_recv(sock, req_name)
    if "Err" in resp:
        notify(f"❌ niri 返回错误：{resp['Err']}")
        sys.exit(1)
    return resp["Ok"][req_name]


def do_action(sock, action_name, **kwargs):
    """
    执行一个 niri action。

    构建的 IPC 消息格式：
      {"Action": {"<动作名>": {<参数>}}}

    例如 FocusWindow(id=123) 发送：
      {"Action": {"FocusWindow": {"id": 123}}}

    例如 SetColumnWidth(change={"SetProportion": 50.0}) 发送：
      {"Action": {"SetColumnWidth": {"change": {"SetProportion": 50.0}}}}
    """
    msg = {"Action": {action_name: kwargs}}
    send_and_recv(sock, msg)


def parse_proportion_pattern(pattern):
    """
    解析比例模式字符串，返回每列比例值列表（值域 0~1）。

    支持的模式：
      "even" / "1/2"          → [50%, 50%]
      "2/3_1/3"              → [66.7%, 33.3%]
      "1/3_2/3"              → [33.3%, 66.7%]
      "1/3_1/3_1/3"          → [33.3%, 33.3%, 33.3%]
      "60_40"                → [60%, 40%]（自定义百分比）
    多于预设列数时，比例循环使用（奇偶列模式会自动延续）。
    """
    known = {
        "even": [0.5, 0.5],
        "1/2": [0.5, 0.5],
        "2/3_1/3": [2.0 / 3, 1.0 / 3],
        "1/3_2/3": [1.0 / 3, 2.0 / 3],
        "1/3_1/3_1/3": [1.0 / 3, 1.0 / 3, 1.0 / 3],
        "1/4_3/4": [0.25, 0.75],
        "3/4_1/4": [0.75, 0.25],
    }
    if pattern in known:
        return known[pattern]

    # 尝试解析自定义格式
    try:
        parts = pattern.split("_")
        values = []
        for p in parts:
            if "/" in p:
                a, b = p.split("/")
                values.append(float(a) / float(b))
            else:
                values.append(float(p) / 100.0)
        if values:
            return values
    except Exception:
        pass

    notify(f"❌ 无法解析模式 '{pattern}'")
    sys.exit(1)


def main():
    # ======================== 参数解析 ========================
    pattern = "even"
    if len(sys.argv) > 1:
        if sys.argv[1] == "--pattern" and len(sys.argv) > 2:
            pattern = sys.argv[2]
        else:
            pattern = sys.argv[1]

    proportions = parse_proportion_pattern(pattern)

    # ======================== 连接 niri IPC ========================
    skt_path = get_socket_path()
    sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
    sock.connect(skt_path)

    # ======================== 找出当前工作区 ========================
    workspaces = request(sock, "Workspaces")
    focused_wspace_id = None
    for ws in workspaces:
        if ws.get("is_focused"):
            focused_wspace_id = ws["id"]
            break
    if focused_wspace_id is None:
        notify("❌ 找不到当前聚焦的工作区")
        sys.exit(1)

    # ======================== 按列分组当前工作区的窗口 ========================
    # 每个窗口的 layout.pos_in_scrolling_layout = [列索引, 行索引]
    # 例如 [0, 0] = 第一列第一行, [1, 0] = 第二列第一行
    all_windows = request(sock, "Windows")
    columns = defaultdict(list)
    for win in all_windows:
        if win.get("workspace_id") != focused_wspace_id:
            continue
        if win.get("is_floating"):
            continue
        layout = win.get("layout", {})
        pos = layout.get("pos_in_scrolling_layout")
        if pos is None:
            continue
        col_idx = pos[0]
        columns[col_idx].append(win)

    if not columns:
        notify("ℹ️ 当前工作区没有平铺窗口")
        sys.exit(0)

    sorted_cols = sorted(columns.keys())

    # ======================== 记住原始焦点 ========================
    # 脚本最后会恢复焦点，不改变用户的操作位置
    focused = request(sock, "FocusedWindow")
    orig_focus_id = focused.get("id")

    # ======================== 逐列设置等比宽度 ========================
    # SetColumnWidth 只对当前聚焦的列生效，所以需要逐列 FocusWindow
    for i, col_idx in enumerate(sorted_cols):
        first_win = columns[col_idx][0]
        target_id = first_win["id"]

        # 聚焦该列的第一个窗口
        do_action(sock, "FocusWindow", id=target_id)

        # 取该列的目标比例，循环使用
        # （例如 pattern=2/3_1/3 时：列0=66.7%, 列1=33.3%, 列2=66.7%, ...）
        pct = proportions[i % len(proportions)] * 100

        # 设回等比宽度（关键！把固定像素值恢复成动态比例）
        do_action(sock, "SetColumnWidth", change={"SetProportion": pct})

    # ======================== 恢复焦点，同时保持视口正确 ========================
    # 逐列 FocusWindow 会导致 niri 的滚动视口跟随焦点偏移。
    # 如果最后聚焦的是第二列，视口就卡在第二列，第一列滚到屏幕左边外面去了。
    #
    # 修复策略（三步，无视觉闪烁）：
    #   1. FocusColumn(0)     → 把视口拉到第一列，确保所有列在屏幕内
    #   2. CenterVisibleColumns → 居中可见列（不改变焦点），消除偏移
    #   3. FocusWindow(orig)  → 恢复原始焦点，此时窗口已在可见区域，不触发滚动

    do_action(sock, "FocusColumn", index=0)           # 1. 视口拉到列0（焦点暂移）
    do_action(sock, "CenterVisibleColumns")            # 2. 居中，列填满屏幕
    if orig_focus_id is not None:
        do_action(sock, "FocusWindow", id=orig_focus_id)   # 3. 恢复焦点（不滚动）

    sock.close()

    # ======================== 桌面通知反馈 ========================
    parts = []
    for i in range(len(sorted_cols)):
        p = proportions[i % len(proportions)]
        parts.append(f"列{i+1} {p*100:.0f}%")
    notify(f"✓ 已重置 {len(sorted_cols)} 列")


if __name__ == "__main__":
    main()
